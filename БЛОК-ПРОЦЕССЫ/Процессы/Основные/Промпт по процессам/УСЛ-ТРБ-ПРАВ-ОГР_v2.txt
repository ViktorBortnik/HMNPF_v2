Промпт на создание таблицы условий, правил, ограничений и требований бизнес-процесса
Назначение таблицы
Создай таблицу для систематизации условий, правил, ограничений и требований бизнес-процесса в формате Excel/Markdown. Таблица предназначена для документирования нормативных, временных, ресурсных и технологических ограничений процесса.

Структура таблицы
Заголовки столбцов:

Категория - классификация по типу элемента:

УСЛОВИЯ

ПРАВИЛА

ТРЕБОВАНИЯ

ОГРАНИЧЕНИЯ

Описание - краткое содержательное описание условия/правила/требования/ограничения (1-2 предложения)

Нормативный источник - ссылка на регулирующий документ:

Федеральные законы (например: ФЗ-75 ст.13)

Постановления (например: Постановление 344-П п.24)

Региональные законы (например: Закон ХМАО №81-ОЗ ст.6)

Внутренние регламенты

Технические ограничения

Временное ограничение - временные рамки или SLA:

Конкретные сроки (≤ 1 рабочий день, ≤ 3 рабочих дня)

Периодичность (ежедневно, ежемесячно)

Константы (постоянно, в день события)

Прочерк (-) если не применимо

Номер задачи - коды задач, к которым применяется:

Формат: ПРЦ[NN].[NN].ЗДЧ[NN]

Можно указывать диапазоны (ПРЦ02.01.ЗДЧ01-10)

Можно указывать группы (Все задачи)

Номер артефакта - коды артефактов, связанных с элементом:

Формат: АРТ[NN].[NN].[NN]

Множественные через запятую (АРТ02.01.11, АРТ02.01.12)

Можно указывать группы (Все АРТ02.01.*)

Можно ссылаться на справочники (Реестр договоров НПО)

Прочерк (-) если не применимо

Правила заполнения
УСЛОВИЯ
Описывают предварительные требования для начала или выполнения процесса:

Наличие документов/данных

Состояние договоров/счетов

Соответствие параметров

Выполнение предварительных этапов

ПРАВИЛА
Определяют порядок действий и методологию выполнения:

Алгоритмы обработки

Методы учета и классификации

Процедуры выполнения операций

Стандарты ведения документооборота

ТРЕБОВАНИЯ
Устанавливают обязательные параметры качества и производительности:

Временные SLA (Service Level Agreement)

Целевые показатели KPI (%, количество)

Обязательные действия

Критерии точности и полноты

ОГРАНИЧЕНИЯ
Фиксируют внешние и внутренние лимиты:

По времени: максимальные сроки, временные рамки

По ресурсам: персонал, системы, технологии, интеграции

По компетенциям: требуемые знания и навыки

По нормативам: законодательные и регуляторные рамки

По объемам: лимиты, квоты, пропускная способность

Форматирование
Группировка:

Первая строка каждой категории содержит только название категории в столбце "Категория"

Остальные ячейки строки-заголовка пустые

Строка выделяется жирным шрифтом

Детализация:

Для ограничений используй подкатегории через двоеточие: "По времени:", "По ресурсам:", "По компетенциям:", "По нормативам:"

Это улучшает навигацию при большом количестве элементов

Временные ограничения:

Используй математические знаки: ≤, ≥, <, >, =

Указывай единицы измерения (рабочий день, календарный день, час)

Используй дефисы для несущественных элементов (-)

Ссылки на нормативы:

Указывай конкретные статьи и пункты

Для множественных ссылок используй сокращения (пп.3-4, 24-26)

Различай уровни: федеральные, региональные, внутренние

Контекст применения
Таблица используется для:

Документирования бизнес-процессов НПФ и финансовых организаций

Аудита соответствия нормативным требованиям

Проектирования автоматизации в системе 1С

BPMN-моделирования с учетом ограничений

Обучения персонала и контроля компетенций

Выявления узких мест и рисков процесса

Планирования ресурсов и SLA
